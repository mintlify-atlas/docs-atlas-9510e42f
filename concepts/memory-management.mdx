FreeRTOS provides its own heap rather than using the C standard library's `malloc()` and `free()` directly. The reasons are practical: bare-metal targets often lack an operating system to back standard malloc, the standard allocator is typically not thread-safe, and embedded systems require predictable allocation timing and controlled fragmentation behaviour.

The FreeRTOS heap is exposed through two functions declared in `portable.h`:

```c
void * pvPortMalloc( size_t xWantedSize );
void   vPortFree( void * pv );
```

All kernel objects — tasks, queues, semaphores, event groups, timers, and stream buffers — call `pvPortMalloc()` internally when created with the dynamic allocation variants of their creation functions.

## configTOTAL_HEAP_SIZE

For heap_1, heap_2, and heap_4, the entire heap is a statically declared byte array:

```c
/* portable/MemMang/heap_1.c — the heap array */
static uint8_t ucHeap[ configTOTAL_HEAP_SIZE ];
```

The size of this array is controlled by `configTOTAL_HEAP_SIZE` in `FreeRTOSConfig.h`:

```c
/* Sets the total size of the FreeRTOS heap in bytes. */
#define configTOTAL_HEAP_SIZE    4096
```

This value must be tuned for your application. It appears in the `.bss` section and consumes RAM even if most of it is never allocated at runtime.

## Heap implementations

FreeRTOS ships five heap implementations in `portable/MemMang/`. Include exactly one in your build.

### heap_1 — allocate only, never free

```
The simplest possible implementation of pvPortMalloc(). Note that this
implementation does NOT allow allocated memory to be freed again.
```
*(from `portable/MemMang/heap_1.c`)*

heap_1 maintains a single index (`xNextFreeByte`) into the `ucHeap` array and advances it on each allocation. `vPortFree()` is a no-op.

Use heap_1 when your application creates all tasks and objects at startup and never deletes them. It is deterministic, has zero fragmentation, and is trivially auditable.

### heap_2 — best-fit allocation with free, no coalescing

```
A sample implementation of pvPortMalloc() and vPortFree() that permits
allocated blocks to be freed, but does not combine adjacent free blocks
into a single larger block (and so will fragment memory). See heap_4.c for
an equivalent that does combine adjacent blocks into single larger blocks.
```
*(from `portable/MemMang/heap_2.c`)*

heap_2 maintains a linked list of free blocks sorted by size. `pvPortMalloc()` uses a best-fit strategy: it selects the smallest free block that satisfies the request. When a block is freed it is returned to the free list but not merged with adjacent free blocks.

<Warning>
heap_2 can fragment over time because freed blocks are never coalesced. After many allocate/free cycles of varying sizes, the heap may consist of many small free blocks, none of which is large enough to satisfy a new request — even if the total free space would be sufficient. For most new applications, heap_4 is the better choice.
</Warning>

### heap_3 — wraps standard malloc/free

heap_3 wraps the compiler's `malloc()` and `free()` with a scheduler suspension to make them thread-safe. The heap size is controlled by the linker, not by `configTOTAL_HEAP_SIZE`.

<Note>
heap_3 delegates all allocation behaviour to the C runtime. Thread safety is provided by suspending the scheduler around each call, not by a mutex, so interrupt-level allocations are not supported. The determinism and fragmentation characteristics are entirely determined by your toolchain's malloc implementation. Only use heap_3 if you have a specific reason to use the standard allocator, such as compatibility with third-party code that calls `malloc()` directly.
</Note>

### heap_4 — first-fit with coalescing (recommended)

```
A sample implementation of pvPortMalloc() and vPortFree() that combines
(coalescences) adjacent memory blocks as they are freed, and in so doing
limits memory fragmentation.
```
*(from `portable/MemMang/heap_4.c`)*

heap_4 maintains a singly linked list of free blocks. `pvPortMalloc()` walks the list and returns the first block large enough for the request (first-fit). When `vPortFree()` is called, the freed block is inserted back into the free list and any adjacent free blocks are merged into a single larger block (coalescing).

<Tip>
heap_4 is the right choice for most projects. It allows dynamic allocation and deallocation, controls fragmentation through coalescing, and does not depend on the C runtime. Enable the optional heap protector (`configENABLE_HEAP_PROTECTOR = 1`) to detect pointer corruption in heap block headers.
</Tip>

### heap_5 — heap_4 across multiple non-contiguous memory regions

```
A sample implementation of pvPortMalloc() that allows the heap to be defined
across multiple non-contiguous blocks and combines (coalescences) adjacent
memory blocks as they are freed.
```
*(from `portable/MemMang/heap_5.c`)*

heap_5 has the same first-fit-with-coalescing behaviour as heap_4 but can span multiple physically separate memory regions (e.g. internal SRAM and external PSRAM). You define the regions with `vPortDefineHeapRegions()`, which **must** be called before any call to `pvPortMalloc()` — that means before creating any task, queue, or other kernel object.

```c
/* From portable/MemMang/heap_5.c — HeapRegion_t definition:
 *
 * typedef struct HeapRegion
 * {
 *   uint8_t * pucStartAddress;
 *   size_t    xSizeInBytes;
 * } HeapRegion_t;
 *
 * The array must be terminated with a NULL/zero entry and ordered
 * from lowest to highest start address.
 */
HeapRegion_t xHeapRegions[] =
{
    { ( uint8_t * ) 0x80000000UL, 0x10000 },  /* 64 KB at 0x80000000 */
    { ( uint8_t * ) 0x90000000UL, 0xa0000 },  /* 640 KB at 0x90000000 */
    { NULL, 0 }                                /* Terminates the array. */
};

/* Call before any pvPortMalloc() — i.e. before any kernel object creation. */
vPortDefineHeapRegions( xHeapRegions );
```

## Comparison table

| Feature | heap_1 | heap_2 | heap_3 | heap_4 | heap_5 |
|---------|--------|--------|--------|--------|--------|
| Can free memory | No | Yes | Yes | Yes | Yes |
| Coalesces free blocks | N/A | No | N/A | Yes | Yes |
| Fragmentation risk | None | High | Toolchain-dependent | Low | Low |
| Allocation strategy | Linear bump | Best-fit | stdlib | First-fit | First-fit |
| Multiple memory regions | No | No | No | No | Yes |
| Depends on stdlib | No | No | Yes | No | No |
| Deterministic timing | Yes | Yes | No | Yes | Yes |
| Requires init call | No | No | No | No | Yes (`vPortDefineHeapRegions`) |
| Best for | Static systems | Rarely | Toolchain integration | Most projects | Multi-region targets |

## Heap monitoring

### Free heap size

```c
/* Returns the total free heap space in bytes (sum of all free blocks). */
size_t xPortGetFreeHeapSize( void );

/* Returns the minimum free heap size ever recorded since boot.
 * Useful for sizing configTOTAL_HEAP_SIZE. */
size_t xPortGetMinimumEverFreeHeapSize( void );
```

These functions are available for all heap implementations except heap_3, where the C runtime manages the heap.

### Detailed heap statistics (heap_4 and heap_5 only)

`vPortGetHeapStats()` populates a `HeapStats_t` structure with detailed information:

```c
/* From include/portable.h */
typedef struct xHeapStats
{
    size_t xAvailableHeapSpaceInBytes;      /* Total free bytes (sum of all free blocks). */
    size_t xSizeOfLargestFreeBlockInBytes;  /* Largest single free block. */
    size_t xSizeOfSmallestFreeBlockInBytes; /* Smallest single free block. */
    size_t xNumberOfFreeBlocks;             /* Number of separate free blocks. */
    size_t xMinimumEverFreeBytesRemaining;  /* Minimum total free bytes ever recorded. */
    size_t xNumberOfSuccessfulAllocations;  /* Total successful pvPortMalloc() calls. */
    size_t xNumberOfSuccessfulFrees;        /* Total successful vPortFree() calls. */
} HeapStats_t;
```

```c
void vPortGetHeapStats( HeapStats_t * pxHeapStats );
```

### Example: checking heap usage after initialisation

```c
#include "FreeRTOS.h"
#include "task.h"

static void vHeapCheckTask( void * pvParameters )
{
    HeapStats_t xHeapStats;

    for( ;; )
    {
        /* Get detailed statistics (heap_4 / heap_5 only). */
        vPortGetHeapStats( &xHeapStats );

        /* Log key metrics — replace with your platform's logging. */
        logPrintf( "Free heap:         %u bytes\r\n",
                   ( unsigned ) xHeapStats.xAvailableHeapSpaceInBytes );
        logPrintf( "Largest free block:%u bytes\r\n",
                   ( unsigned ) xHeapStats.xSizeOfLargestFreeBlockInBytes );
        logPrintf( "Min ever free:     %u bytes\r\n",
                   ( unsigned ) xHeapStats.xMinimumEverFreeBytesRemaining );
        logPrintf( "Alloc calls:       %u\r\n",
                   ( unsigned ) xHeapStats.xNumberOfSuccessfulAllocations );
        logPrintf( "Free calls:        %u\r\n",
                   ( unsigned ) xHeapStats.xNumberOfSuccessfulFrees );

        /* Also available without HeapStats_t: */
        logPrintf( "xPortGetFreeHeapSize:           %u\r\n",
                   ( unsigned ) xPortGetFreeHeapSize() );
        logPrintf( "xPortGetMinimumEverFreeHeapSize:%u\r\n",
                   ( unsigned ) xPortGetMinimumEverFreeHeapSize() );

        vTaskDelay( pdMS_TO_TICKS( 5000 ) );
    }
}
```

Call `xPortGetMinimumEverFreeHeapSize()` after your application has been running under realistic load for some time. The result tells you the worst-case heap consumption — use it to set `configTOTAL_HEAP_SIZE` with an appropriate safety margin.

## Static allocation as an alternative

Dynamic allocation can be eliminated entirely. Every FreeRTOS object that has a `xCreate()` function also has an `xCreateStatic()` variant that accepts caller-provided buffers. Enable this with:

```c
/* FreeRTOSConfig.h */
#define configSUPPORT_STATIC_ALLOCATION     1
#define configSUPPORT_DYNAMIC_ALLOCATION    0  /* Optional: disable dynamic alloc. */
```

Static allocation examples:

```c
/* Static task */
static StackType_t  xTaskStack[ 256 ];
static StaticTask_t xTaskBuffer;
xTaskCreateStatic( vMyTask, "T", 256, NULL, 1, xTaskStack, &xTaskBuffer );

/* Static queue (holds 10 uint32_t items) */
static uint8_t         xQueueStorage[ 10 * sizeof( uint32_t ) ];
static StaticQueue_t   xQueueBuffer;
QueueHandle_t xQueue = xQueueCreateStatic( 10, sizeof( uint32_t ),
                                           xQueueStorage, &xQueueBuffer );
```

Static allocation is the right approach for safety-critical systems where every memory allocation must be traceable to a fixed-size buffer and runtime allocation failures are unacceptable.
