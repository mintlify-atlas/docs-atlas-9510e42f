---
title: "Software Timers"
description: "One-shot and auto-reload software timers that execute callbacks in the timer task context"
---

Software timers let you execute a callback function at a future time, or repeatedly at a fixed interval, without dedicating a task to the purpose. All timer callbacks run in the context of the RTOS daemon task (also called the timer service task), which FreeRTOS creates automatically when `configUSE_TIMERS` is set to `1`.

Because the daemon task is a standard FreeRTOS task, its stack and priority are configurable, and it competes for CPU time like any other task.

## Enabling software timers

Add the following to `FreeRTOSConfig.h` and include `timers.c` in your build:

```c FreeRTOSConfig.h
/* Required: enable the software timer feature */
#define configUSE_TIMERS                1

/* Priority of the timer daemon task. Typically set to the highest priority
 * so that timer callbacks are not starved by application tasks. */
#define configTIMER_TASK_PRIORITY       ( configMAX_PRIORITIES - 1 )

/* Depth of the timer command queue. Commands (start, stop, reset, etc.)
 * are posted here and processed by the daemon task. */
#define configTIMER_QUEUE_LENGTH        10

/* Stack size for the timer daemon task, in words (not bytes). */
#define configTIMER_TASK_STACK_DEPTH    configMINIMAL_STACK_SIZE
```

<Info>
`timers.c` must be included in your build when `configUSE_TIMERS` is set to `1`. It is located in the FreeRTOS kernel source directory alongside `tasks.c`.
</Info>

## Timer modes

| Mode | `xAutoReload` value | Behaviour |
|---|---|---|
| One-shot | `pdFALSE` | The callback fires once. The timer enters the dormant state after expiry. |
| Auto-reload | `pdTRUE` | The callback fires repeatedly at the configured period until the timer is stopped or deleted. |

## Core types

```c
/* Handle type for software timers. */
typedef struct tmrTimerControl * TimerHandle_t;

/* Prototype that all timer callback functions must match. */
typedef void (* TimerCallbackFunction_t)( TimerHandle_t xTimer );
```

## Creating a timer

<Steps>
  <Step title="Define the callback function">
    The callback must match `TimerCallbackFunction_t`. It receives the handle of the timer that expired.

    ```c
    void vMyTimerCallback( TimerHandle_t xTimer )
    {
        /* Do work here. Must not block. */
        uint32_t ulTimerID = ( uint32_t ) pvTimerGetTimerID( xTimer );
    }
    ```
  </Step>

  <Step title="Create the timer">
    Use `xTimerCreate()` for dynamic allocation or `xTimerCreateStatic()` when you need deterministic memory usage.

    ```c
    /* Dynamic allocation */
    TimerHandle_t xTimerCreate(
        const char * const        pcTimerName,
        const TickType_t          xTimerPeriodInTicks,
        const BaseType_t          xAutoReload,
        void * const              pvTimerID,
        TimerCallbackFunction_t   pxCallbackFunction );

    /* Static allocation — caller supplies the StaticTimer_t buffer */
    TimerHandle_t xTimerCreateStatic(
        const char * const        pcTimerName,
        const TickType_t          xTimerPeriodInTicks,
        const BaseType_t          xAutoReload,
        void * const              pvTimerID,
        TimerCallbackFunction_t   pxCallbackFunction,
        StaticTimer_t *           pxTimerBuffer );
    ```

    Timers are created in the **dormant** state. They do not start until you call `xTimerStart()` or a related function.

    ```c
    TimerHandle_t xLedTimer = xTimerCreate(
        "LedTimer",           /* Name for debugging only */
        pdMS_TO_TICKS( 500 ), /* Period: 500 ms */
        pdTRUE,               /* Auto-reload */
        ( void * ) 0,         /* Timer ID */
        vLedTimerCallback
    );

    if( xLedTimer == NULL )
    {
        /* Insufficient heap. Increase configTOTAL_HEAP_SIZE. */
    }
    ```
  </Step>

  <Step title="Start the timer">
    `xTimerStart()` sends a command to the timer command queue. The second argument is the maximum time to wait if the queue is full.

    ```c
    /* Start before the scheduler: block time is ignored */
    if( xTimerStart( xLedTimer, 0 ) != pdPASS )
    {
        /* Command queue full — increase configTIMER_QUEUE_LENGTH */
    }

    vTaskStartScheduler();
    ```
  </Step>
</Steps>

## Millisecond conversion

Use `pdMS_TO_TICKS()` to convert milliseconds to tick counts without depending on `configTICK_RATE_HZ` at the call site:

```c
TickType_t xPeriod = pdMS_TO_TICKS( 1000 ); /* 1 second */
```

## Controlling a timer

All control functions send a command to the timer command queue and return `pdPASS` on success or `pdFAIL` if the queue was full within the specified block time.

```c
/* Start (or restart if already active) */
BaseType_t xTimerStart( TimerHandle_t xTimer, TickType_t xTicksToWait );

/* Stop — timer enters dormant state */
BaseType_t xTimerStop( TimerHandle_t xTimer, TickType_t xTicksToWait );

/* Re-evaluate the expiry time from now.
 * Equivalent to xTimerStart() if the timer is dormant. */
BaseType_t xTimerReset( TimerHandle_t xTimer, TickType_t xTicksToWait );

/* Change the period. Also starts the timer if it is dormant. */
BaseType_t xTimerChangePeriod(
    TimerHandle_t xTimer,
    TickType_t    xNewPeriod,
    TickType_t    xTicksToWait );

/* Delete the timer and free its resources */
BaseType_t xTimerDelete( TimerHandle_t xTimer, TickType_t xTicksToWait );
```

## FromISR variants

When calling timer APIs from an interrupt service routine, use the `FromISR` versions. These do not accept a block time; they either post immediately or fail.

```c
BaseType_t xTimerStartFromISR(
    TimerHandle_t xTimer,
    BaseType_t *  pxHigherPriorityTaskWoken );

BaseType_t xTimerStopFromISR(
    TimerHandle_t xTimer,
    BaseType_t *  pxHigherPriorityTaskWoken );

BaseType_t xTimerResetFromISR(
    TimerHandle_t xTimer,
    BaseType_t *  pxHigherPriorityTaskWoken );

BaseType_t xTimerChangePeriodFromISR(
    TimerHandle_t xTimer,
    TickType_t    xNewPeriod,
    BaseType_t *  pxHigherPriorityTaskWoken );
```

After calling any `FromISR` function, pass `pxHigherPriorityTaskWoken` to `portYIELD_FROM_ISR()`:

```c
void vMyISR( void )
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    xTimerResetFromISR( xMyTimer, &xHigherPriorityTaskWoken );

    portYIELD_FROM_ISR( xHigherPriorityTaskWoken );
}
```

## Timer ID and context data

Every timer stores a `void *` identifier set at creation time or updated with `vTimerSetTimerID()`. This is useful when a single callback handles multiple timers.

```c
/* Retrieve the ID */
void * pvTimerGetTimerID( const TimerHandle_t xTimer );

/* Update the ID at runtime */
void vTimerSetTimerID( TimerHandle_t xTimer, void * pvNewID );
```

<Tip>
Store a pointer to a state structure in the timer ID. The callback casts it back to the correct type, giving each logical timer its own private state without needing a separate lookup table.
</Tip>

## Query functions

```c
/* Returns the name string passed to xTimerCreate() */
const char * pcTimerGetName( TimerHandle_t xTimer );

/* Returns the configured period in ticks */
TickType_t xTimerGetPeriod( TimerHandle_t xTimer );

/* Returns pdFALSE if the timer is dormant, non-zero if active */
BaseType_t xTimerIsTimerActive( TimerHandle_t xTimer );

/* Returns the tick count at which the timer will next expire.
 * Only valid while the timer is active. */
TickType_t xTimerGetExpiryTime( TimerHandle_t xTimer );
```

## Example: LED blink with an auto-reload timer

```c
#include "FreeRTOS.h"
#include "timers.h"

static TimerHandle_t xBlinkTimer;

static void vBlinkCallback( TimerHandle_t xTimer )
{
    /* Toggle the LED — runs in the timer daemon task context */
    vToggleLed( LED_0 );
}

void vStartBlinkTask( void )
{
    xBlinkTimer = xTimerCreate(
        "Blink",
        pdMS_TO_TICKS( 500 ), /* Toggle every 500 ms */
        pdTRUE,               /* Auto-reload */
        ( void * ) 0,
        vBlinkCallback
    );

    configASSERT( xBlinkTimer != NULL );

    xTimerStart( xBlinkTimer, 0 );
}
```

## Example: timeout detection with a one-shot timer

Use `xTimerReset()` to restart the timeout countdown each time activity is detected. If the timer expires without being reset, the callback fires to signal an idle condition.

```c
#include "FreeRTOS.h"
#include "timers.h"

#define INACTIVITY_TIMEOUT_MS  5000UL

static TimerHandle_t xTimeoutTimer;

/* Called when no activity has been seen for INACTIVITY_TIMEOUT_MS. */
static void vTimeoutCallback( TimerHandle_t xTimer )
{
    /* No activity for 5 seconds — handle timeout */
    vHandleInactivity();
}

void vInitTimeoutTimer( void )
{
    xTimeoutTimer = xTimerCreate(
        "Timeout",
        pdMS_TO_TICKS( INACTIVITY_TIMEOUT_MS ),
        pdFALSE,    /* One-shot */
        ( void * ) 0,
        vTimeoutCallback
    );

    configASSERT( xTimeoutTimer != NULL );
    xTimerStart( xTimeoutTimer, 0 );
}

/* Call this from any task whenever activity is detected. */
void vOnActivityDetected( void )
{
    /* Reset restarts the one-shot countdown from now.
     * Also starts the timer if it was dormant. */
    xTimerReset( xTimeoutTimer, pdMS_TO_TICKS( 10 ) );
}
```

<Warning>
Timer callbacks must not block. Do not call `vTaskDelay()`, `xQueueReceive()` with a non-zero block time, or any other blocking API from within a timer callback. All timer callbacks share the single daemon task, so a blocked callback prevents every other pending timer from firing.

When stopping or resetting a timer from within its own callback, use a block time of `0` — passing a non-zero block time from a callback risks deadlocking the daemon task against itself.
</Warning>
